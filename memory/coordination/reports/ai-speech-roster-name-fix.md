**Prerequisite:** `ai-speech-name-detection-harness` (Accepted) — detector assertions untouched.

## Summary

No displayed AI speech can now reference a player outside the current game
roster. The fix is layered exactly along the CONFIRMED hypotheses of the
evidence report: the AIWolf corpus is sanitized offline by a deterministic,
re-runnable script (H1); historical log lines are entity-scrubbed before they
enter any LLM prompt (H2/H3); the translation flow enumerates protected
referents in its prompt and verifies the response, falling back to the
original text on violation (H5); and a final-boundary roster guard
(`src/services/rosterGuard.ts`) checks every displayed speech — LLM, library,
wolf-chat, vote-reason, and hardcoded-fallback layers — applying unambiguous
structural repair or falling through to the next layer / a nameless
roster-safe line (H8, always in scope). A foreign name is never mapped onto a
real playerId. The guard reuses the harness detector by importing it, so the
guard and the audit can never disagree.

## Fix invariants — test backing (all 8)

1. **playerId is internal identity** — `rosterGuard.test.ts` "invariant 1": a roster-contradicting `3号 Luna` pair repairs to the structural `3号`; repair introduces no seat ref the input lacked.
2. **Model receives only the canonical roster** — `aiOrchestrator.test.ts` prompt-hygiene test: polluted log line reaches the prompt entity-free with roster seat refs intact (plus the harness's own prompt assertion).
3. **Corpus sanitized, never wrong-playerId-masked** — `speechLibrary.test.ts` corpus scan: zero `Agent[XX]`/AIWolf names in all six shipped pools; placeholders are seat-neutral by construction.
4. **Translation protects referents** — `translationService.test.ts` H5 block: rewritten seat → original returned; introduced name/Agent ref → original returned; preserved referents → accepted; prompt enumerates protected tokens.
5. **Cache isolation** — `translationService.test.ts`: same log id, different language → separate fetches (key `logId:language`); per-id dedup tests retained.
6. **Zero out-of-roster refs in final text; never display bad text** — `rosterGuard.test.ts` repair suite + `aiOrchestrator.test.ts` polluted-LLM day-speech/wolf-chat/vote-reason tests (final text detector-clean); nameless fallback lines detector-clean against any roster including empty.
7. **No game-rule changes; LLM never decides legality** — `aiOrchestrator.test.ts`: out-of-roster LLM vote target rejected, fallback stays inside the valid set; guard never alters `targetId`.
8. **Dev-only diagnostic metadata** — `rosterGuard.test.ts`: `speechDiagnosticsEnabled()` false outside `vite dev`; `emitSpeechDiagnostic` (source/model/fallbackReason/repaired — no prompts, no keys) emits nothing when disabled; orchestrator return shape is exactly `{en, zh}` (metadata never attached to player-visible output).

## Decisions

- **Guard imports the harness detector** (`src/services/rosterGuard.ts` →
  `src/diagnostics/nameDetector.ts` + `aiwolf-entities.json`), as the card's
  guidance explicitly allows (import-only, unmodified). This guarantees
  guard/audit agreement. Side effect: the header comment in `nameDetector.ts`
  ("nothing here is imported by product code") is now stale; the file could
  not be touched under this card's allowed changes — flagged for the
  coordinator as a one-line follow-up.
- **Repair policy:** foreign entities → seat-neutral placeholders (never a
  playerId); contradicting seat-name pair → seat kept, name dropped (the seat
  is the structural reference); out-of-roster seats → neutral placeholder
  (never remapped to another seat). Repair accepted only if the detector
  re-run is clean, else the layer is rejected.
- **Translation check is introduction-only** (new seats/names rejected;
  dropped referents tolerated): the pre-existing service contract and its
  tests accept summarizing proxy responses (e.g. `'翻译结果'` for a source
  containing `3号`), which must not be broken, and a dropped seat cannot point
  at a wrong player. The translation prompt still instructs full preservation.
- **`speechLibrary.ts` unchanged:** the invariant is owned by the data layer
  (sanitized corpus, enforced by `speech-corpus-name-audit.mjs` + a unit
  test) with the orchestrator-side guard as the runtime boundary — smaller
  diff than a pick-time sanitizer and no double ownership.
- Corpus facts simplifying the fix: the corpus contains no seat refs above 5
  (measured), so sanitized library picks cannot produce out-of-roster seats on
  any supported board; the guard still covers that class for LLM output.

## Residual risks

- **H8 live-model hallucination coverage depends on the output guard, not
  offline proof** (explicitly per card): a live model could emit a name
  outside the 69-entity list + 12 product names + seat syntaxes; such text is
  undetectable by the harness detector by construction (inherited limitation,
  also noted in the harness report). Everything detectable is now blocked at
  the final boundary.
- The 2,514 dropped entries slightly reduce pool variety (pools remain 433+
  entries; day/tag distribution unchanged in kind).
- Placeholder phrasing ("that player" / 那位玩家) is seat-anonymous; a few
  sanitized lines read more vaguely than the originals — cosmetic only.
- `.gitignore` contains `**/*.test.ts` (Netlify-deploy exclusion), so the new
  `src/services/rosterGuard.test.ts` is untracked-ignored; the coordinator
  must `git add -f` it when committing (existing test files are already
  force-tracked).
- The harness samples directory now contains the post-fix zero-violation
  snapshot (deterministic); it is regenerated on every gated audit run.
